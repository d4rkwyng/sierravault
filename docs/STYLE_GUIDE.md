# SierraVault Style Guide

Formatting standards for all content in the vault.

## File Naming

### Game Pages
```
YYYY - Full Game Title.md
```

**Examples:**
- `1984 - King's Quest - Quest for the Crown.md`
- `1989 - Space Quest III - The Pirates of Pestulon.md`
- `1990 - Roberta Williams' King's Quest I - Quest for the Crown.md` (remakes include original name)

**Special prefixes:**
- `CXL - YYYY - Title.md` - Cancelled games
- `TBD - Title.md` - Unreleased/in-development games
- `TBA - Title.md` - Announced but undated games

### Designer/Developer/Publisher Pages
```
Full Name.md
```

**Examples:**
- `Roberta Williams.md`
- `Sierra On-Line.md`
- `Dynamix.md`

### Guide Pages
```
Descriptive Title.md
```

**Examples:**
- `Buying Guide.md`
- `Engine Index.md`

## YAML Frontmatter

### Game Pages

```yaml
---
title: "Game Title"
release_year: 1989
developer: "Sierra On-Line"
designer:
  - "Designer Name"
publisher: "Sierra On-Line"
genre: "Adventure"
platforms: ["MS-DOS", "Amiga", "Macintosh"]
series: "Series Name"
engine: "SCI0"
protagonist: "Character Name"
sierra_lineage: "Core Sierra"
last_updated: "2026-01-30"
description: "Brief 1-2 sentence summary for metadata/SEO"
tags: [1980s, adventure, series-name, designer-name]
composer: ["Composer Name"]  # optional
---
```

**Rules:**
- **Plain text only** - NO wiki links in YAML
- Use `null` for unknown fields, not empty strings
- `last_updated` is auto-generated by scripts
- `description` and `tags` are optional but recommended for SEO
- `composer` is optional, use array format

## Section Structure

### Game Pages (Standard Order)

1. YAML frontmatter
2. `# Title`
3. `<small style="color: gray">Last updated: Month DD, YYYY</small>`
4. `## Overview`
5. `> [!info]- Game Info` (collapsible callout)
6. `## Story Summary`
7. `## Gameplay`
   - `### Interface and Controls`
   - `### Structure and Progression`
   - `### Puzzles and Mechanics`
8. `## Reception`
   - `### Contemporary Reviews`
   - `### Modern Assessment`
9. `## Development`
   - `### Origins` / `### Production`
   - `### Voice Cast` (table, if applicable)
   - `### Technical Achievements`
   - `### Version History` (table)
10. `## Legacy`
    - `### Collections`
11. `## Downloads`
    - `**Purchase / Digital Stores**`
    - `**Download / Preservation**`
    - `**Manuals & Extras**`
12. `## See Also` (series navigation, related pages)
13. `## References`

**Optional sections** (add when relevant):
- `### Technical Specifications` - Under Development
- `### Cut Content` - Under Development
- `### Technical Issues` - Under Development
- `### Easter Eggs and Trivia` - Under Development or Legacy
- `### Voice Cast` - Under Development (table format)
- `### Fan Projects` - Under Legacy

### Non-Narrative Games

Sports, racing, card, puzzle, and educational games skip:
- Story Summary
- Puzzles and Mechanics
- Structure and Progression

### Cancelled Games

Skip (with lower reference threshold of 5+):
- Downloads
- Interface and Controls
- Structure and Progression
- Puzzles and Mechanics

## Reference Formatting

### Inline Citations

```markdown
The game sold over 500,000 copies.[^ref-1]
```

### Reference Definitions

```markdown
## References

[^ref-1]: [Source Title](https://example.com/page) - Brief description of source
[^ref-2]: [Author Name, "Article Title"](https://example.com/article) - Publication, Date
```

**Rules:**
- Sequential numbering (`ref-1`, `ref-2`, etc.)
- No duplicate URLs (consolidate to single reference)
- Minimum 15 references for standard pages
- Minimum 5 references for cancelled/TBD pages

## Wiki Links

### Internal Links

```markdown
[[Page Title]]
[[Page Title|Display Text]]
```

**Rules:**
- Use basename only - NO folder paths
- NO `.md` extension
- Use pipe for different display text

**Correct:**
```markdown
[[Roberta Williams]]
[[1990 - King's Quest V - Absence Makes the Heart Go Yonder|King's Quest V]]
```

**Wrong:**
```markdown
[[Designers/Roberta Williams.md]]
[[Games/King's Quest/1990 - King's Quest V.md]]
```

### Apostrophes in Links

Use standard apostrophe (`'`), not curly (`'`):

```markdown
[[King's Quest V]]    ✓ Correct
[[King's Quest V]]    ✗ Wrong
```

## Callouts

### Game Info (Collapsible)

```markdown
> [!info]- Game Info
> **Developer:** [[Studio Name]][^ref-1]
> **Designer:** [[Designer Name]][^ref-1]
> **Publisher:** Publisher Name[^ref-1]
> **Engine:** Engine Name[^ref-2]
> **Platforms:** Platform list[^ref-3]
> **Release Year:** Year[^ref-1]
> **Series:** Series Name
> **Protagonist:** Character Name
> **Sierra Lineage:** Core Sierra
```

**Rules:**
- Wiki links ARE allowed for Developer, Designer, Publisher links
- Add citations to factual fields

**Additional fields** (use when relevant):
- `**Producer:**` - Executive/line producer
- `**Programmer:**` - Lead programmer  
- `**Artist:**` - Lead artist/art director
- `**Composer:**` - Music composer
- `**Design/Writing:**` - Alternative to Designer when same person wrote story

### Other Callouts

```markdown
> [!note] Title
> Content

> [!warning] Title
> Content

> [!tip] Title
> Content
```

## Spoilers

Protect major plot reveals to preserve the player experience.

### What Counts as a Spoiler

**Always protect:**
- Killer/villain identity reveals (especially mystery games)
- Major character deaths
- Ending twists and surprises
- Betrayals by trusted characters

**Don't protect:**
- Marketing material (game box descriptions)
- General premise or setup
- Gameplay mechanics
- Well-known historical facts about the game

### Spoiler Format

Use collapsible warning callouts (native Obsidian, works on Publish):

```markdown
> [!warning]- Spoiler Title
> The spoiler content goes here.
```

**Examples:**

```markdown
> [!warning]- Major Spoilers
> **Killer Identity:** Bruno from QFG1 is the assassin.
> 
> **Character Fate:** Rakeesh can die depending on choices.
```

```markdown
> [!warning]- Ending Spoiler
> The detective was corrupt all along.
```

**Note:** Inline spoilers (`<span class="spoiler">`) don't work on Obsidian Publish — always use callouts.

### Priority by Genre

| Genre | Spoiler Sensitivity |
|-------|---------------------|
| Mystery (Laura Bow, GK) | **Critical** - killer reveals sacred |
| Horror (Phantasmagoria) | High - twist reveals important |
| Adventure (KQ, SQ, QFG) | Medium - major deaths, endings |
| Action/Sim | Low - gameplay-focused |

## Sierra Lineage Values

Use only these canonical values:

**Core Sierra:**
- `Core Sierra` - Direct Sierra releases (1980-1999)

**Sierra Labels (Subsidiaries):**
- `Sierra Label (Dynamix)` - Dynamix games
- `Sierra Label (Impressions)` - Impressions games
- `Sierra Label (Coktel)` - Coktel Vision games
- `Sierra Label (Papyrus)` - Papyrus games
- `Sierra Label (Bright Star)` - Bright Star games
- `Sierra Label (Synergistic)` - Synergistic Software games
- `Sierra Label (Sierra Attractions)` - Sierra Attractions games
- `Sierra Label (Discovery)` - Discovery games

**Other Relationships:**
- `Sierra Published` - Third-party games published by Sierra
- `Third-Party Published` - Games localized/distributed by Sierra (Silpheed, Zeliard)
- `Acquired Franchise` - Pre-acquisition games from acquired studios

**Post-Sierra/Fan:**
- `Fan Project` - AGDI remakes, fan games
- `Alumni Project` - Projects by Sierra alumni (Hero-U, etc.)
- `Spiritual Successor` - Games inspired by Sierra titles
- `Spinoff` - Derivative products (LSL Casino, etc.)
